🛠️ 万能 Prompt 模板
点击右上角按钮一键复制，适配所有 AI 工具
# 万能 Prompt 公式 你是一个 [角色]，拥有 [相关经验/背景]。 ## 任务 我需要你帮我 [具体任务描述]。 ## 背景信息 [简要上下文，如：我是腾讯 IEG 的新人，正在设计...] ## 输出要求 1. 格式：[表格 / 列表 / 段落 / Markdown] 2. 字数：[约 XXX 字] 3. 风格：[正式 / 口语化 / 专业但不枯燥] 4. 特别注意：[任何特殊约束，如：必须符合广电总局规定] ## 质量自检 请在回答中标注 [事实 / 推测 / 建议] 的区分，并用 FACT 框架自查一遍。
🔗 链式提问策略
第一轮（广度）：帮我列出 [主题] 的所有关键维度。
第二轮（深度）：针对 [维度 A]，请详细展开，给出操作步骤。
第三轮（批判）：你刚才提到的 [某点] 有什么反例或局限性？
第四轮（应用）：基于以上分析，请给我 3 个具体可落地的建议。
⚠️ 常见翻车 & 补救
😭 翻车现场 根本原因 🩹 补救措施
回答太泛，没有针对性 缺少业务上下文约束 加上"聚焦在 XX 业务场景，输出要具体到可以 直接执行"
回答跑偏，不是我想要的 任务描述模糊 "具体来说，我需要的是 XXX，而不是 YYY。请重新回答。"
输出太长，看不下去 没有字数 / 格式限制 "请用 300 字以内 回答，并用 表格 输出"
AI 编造数据 / 幻觉 模型固有缺陷 "请 标注信息来源，不确定的内容请 明确注明【推测】"
🚦 AI 使用安全决策树
🚨 红线：绝对不能喂给 AI 的内容
1. 用户 PII（姓名/电话/地址/身份证）— 违反《个人信息保护法》
2. 未公开商业机密（游戏数值/商业化策略/未发布功能）— 泄露=问责
3. 系统密钥/Token（API Key/数据库密码/内部服务地址）— 被攻破=灾难
4. 人事信息（绩效/薪酬/晋升评审）— 违反内部合规
⚠️ 黄线：需要脱敏后使用的内容
1. 用户访谈记录 → 删除姓名/联系方式，保留行为描述
2. 业务数据 → 只保留趋势/比例，不保留精确值（如：留存率45% ✅，DAU=12,345 ❌）
3. 代码片段 → 删除敏感变量名、IP地址、注释中的业务背景
4. PRD/技术方案 → 删除具体时间点、人员分工、竞品对比细节
✅ 绿线：可以放心使用的内容
1. 公开技术文档（API文档/开源项目README/技术博客）
2. 已脱敏的调研报告（删除用户标识，保留洞察结论）
3. 个人工作总结（不含敏感信息的方法论/心得/反思）
4. 行业标准/竞品公开信息（官网/财报/公开访谈）
💡 安全决策术：每次喂给 AI 之前，问自己："如果这条内容出现在竞争对手手里，会有多糟？"如果答案是"有点糟"，就脱敏。如果是"很糟"，就别喂。
